## 一、交互设计原则

| 原则 | 说明 |
|-----|------|
| **直观** | 用户无需学习即可操作 |
| **反馈** | 每个操作都有即时反馈 |
| **流畅** | 动画平滑，无卡顿 |
| **一致性** | 相同功能使用相同交互模式 |
| **容错** | 操作可撤销，错误可恢复 |

## 二、页面交互流程

### 2.1 首页交互

```
用户进入首页
    ↓
查看顶部品牌区（闪光少年 + slogan）
    ↓
点击「跳绳」卡片 → 跳转到跳绳录制页
    ↓
点击「足球」卡片 → 跳转到足球训练页
    ↓
查看最近训练列表
    ↓
点击某条训练记录 → 跳转到详情报告页
```

### 2.2 跳绳训练交互

```
选择训练模式（计时/定量/自由）
    ↓
点击开始训练
    ↓
倒计时3秒（动画提示）
    ↓
实时显示：计数 + BPM + 时间
    ↓
训练结束（达到目标或手动停止）
    ↓
显示训练结果卡片（动画弹出）
    ↓
点击查看报告 → 跳转到六边形报告页
```

### 2.3 六边形报告交互

```
进入报告页
    ↓
六边形动画绘制（从中心向外展开）
    ↓
数字滚动动画（计数、BPM等）
    ↓
点击「分享」→ 弹出分享选项（微信/保存图片）
    ↓
点击「对比上次」→ 显示进步对比
    ↓
点击训练建议 → 展开详情
```

## 三、动画设计

### 3.1 页面转场

| 转场类型 | 动画效果 | 时长 |
|---------|---------|-----|
| 首页→训练页 | 卡片放大进入 | 300ms |
| 训练页→报告页 | 结果卡片弹出 | 400ms |
| 返回 | 从左侧滑出 | 250ms |

### 3.2 六边形绘制

```
时间轴：
0ms → 中心点出现
100ms → 第一条边（爆发力）
200ms → 第二条边（速度）
300ms → 第三条边（耐力）
400ms → 第四条边（协调性）
500ms → 第五条边（柔韧性）
600ms → 第六条边（稳定性）
600ms → 填充区域渐变
800ms → 数值标签淡入
1000ms → 完成
```

### 3.3 数字滚动

```
效果：从0滚动到目标值
时长：800ms
曲线：easeOut
示例：
    计数：0 → 45 → 89 → 127 → 150
    BPM：-- → 98 → 115 → 120
```

## 四、交互细节

### 4.1 按钮交互

| 状态 | 效果 |
|-----|------|
| 正常 | 原色背景 |
| 按下 | 亮度降低10% |
| 禁用 | 灰色背景，文字变淡 |
| 悬停 | 阴影加深 |

### 4.2 列表交互

| 操作 | 效果 |
|-----|------|
| 点击 | 跳转详情页 |
| 长按 | 弹出操作菜单（删除/分享） |
| 滑动 | 显示删除按钮 |

### 4.3 加载状态

| 场景 | 效果 |
|-----|------|
| 页面加载 | 骨架屏占位 |
| 数据请求 | 加载动画 |
| 视频分析 | 进度条 + 提示文字 |

## 五、异常处理

### 5.1 网络异常

| 场景 | 处理方式 |
|-----|---------|
| 无网络 | 显示离线提示，提供缓存数据 |
| 请求失败 | 显示错误提示，提供重试按钮 |

### 5.2 权限异常

| 场景 | 处理方式 |
|-----|---------|
| 相机权限拒绝 | 引导用户开启权限 |
| 存储权限拒绝 | 提示影响功能 |

### 5.3 数据异常

| 场景 | 处理方式 |
|-----|---------|
| 视频解析失败 | 提示重新拍摄 |
| AI分析失败 | 提示重试 |
